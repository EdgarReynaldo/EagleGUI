◆ DIALOG_RETURN_VALUE

enum DIALOG_RETURN_VALUE

Bitfield values for messages returned to a dialog from the Update() and CheckInput() functions.

Valid values are any bitwise OR'ed combination of the following flags :

Enumerator
DIALOG_OKAY 

Return value used to specify no messages Also used to signify that there are no messages for the WidgetHandler::TakeNextMessage function.

DIALOG_CLOSE 

Return value used to tell our widget handler to close the dialog.

DIALOG_REDRAW_ALL 

For when the background needs to be cleared / redrawn.

DIALOG_INPUT_USED 

Play nice, a WidgetHandler will stop checking other widgets input after this is received.

DIALOG_TAKE_FOCUS 

Add this flag if you want to take the focus.

DIALOG_DISABLED 

Returned when you try to update or check the inputs of a disabled dialog.

DIALOG_REMOVE_ME 

Used for popups, so they can be removed timely.

◆ WIDGET_ZORDER_PRIORITY

enum WIDGET_ZORDER_PRIORITY

This enum provides a small range of z depth for widgets, to enable layering.

WIDGET_ZORDER_PRIORITY controls the z order for widgets used by a WidgetHandler. Widgets with a higher priority appear closer to the viewer and lower priority widgets are sorted beneath them.
